# Kick Assembler - IValue interface overview: values represent assembler values (numbers, strings, booleans). Methods include getInt(), getDouble(), getString(), getBoolean(), getList() returning List<IValue>, and predicate methods hasIntRepresentation(), hasDoubleRepresentation(), hasStringRepresentation(), hasBooleanRepresentation(), hasListRepresentation().

It can either be passed as argument to the plugin or created by the plugin and returned as a result. Use the
'createMemoryBlock' in the IEngine interface to create new memory blocks.

Table 17.3. IMemoryBlock Interface
Method

Description

int getStartAddress()

The start address of the memory block.
