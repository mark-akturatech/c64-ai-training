# ********* - Table 5.1 — BAM (Block Availability Map) layout for the 1540/1541 disk. Covers Track 18 Sector 0 bytes: bytes 0–1 pointing to first directory block, byte 2 (ASCII 'A' for 4040 format), byte 3 (DOS reserved/null flag), and bytes 4–143 containing the bit map of available blocks for tracks 1–35 (each bit represents a block; 1 = available, 0 = not available).


                 Table 5.1: 1540/1541 BAM FORMAT
 +-----------------------------------------------------------------+
 | Track 18, Sector 0.                                             |
 +-------+----------+----------------------------------------------+
 | BYTE  | CONTENTS |                DEFINITION                    |
 +-------+----------+----------------------------------------------+
 | 0,1   | 18,01    | Track and sector of first directory block.   |
 +-------+----------+----------------------------------------------+
 | 2     | 65       | ASCII character A indicating 4040 format.    |
 +-------+----------+----------------------------------------------+
 | 3     | 0        | Null flag for future DOS use.                |
 +-------+----------+----------------------------------------------+
 | 4-143 |          | Bit map of available blocks for tracks 1-35. |
 +-------+----------+----------------------------------------------+
 | *1 = available block                                            |
 |  0 = block not available                                        |
 |      (each bit represents one block)                            |
 +-----------------------------------------------------------------+
