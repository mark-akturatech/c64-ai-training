# BASIC POKE vs Machine Language: placing "A" on screen (CBM ASCII vs screen codes) and a 64MON step‑by‑step

**Summary:** This document explains the difference between CBM ASCII (PRINT ASC) and screen/PETSCII codes used by POKE, demonstrates how to place the letter "A" on the screen using POKE commands, and provides a step-by-step guide to creating a simple machine language program with 64MON that writes a character to screen memory and sets its color.

**CBM ASCII vs Screen Codes (POKE)**

- `PRINT ASC("A")` returns 65 (decimal), which is the CBM/ASCII code for uppercase "A".
- Screen memory (default at $0400 / decimal 1024) uses different character codes (screen/PETSCII codes). In this example, the screen code for the visible "A" is 1, so `POKE 1024,1` writes an "A" glyph to the top-left of the screen.
- Color RAM begins at $D800 (decimal 55296). `POKE 55296,14` sets the color value at the first character cell (the example uses 14).
- The practical effect: `POKE 1024,1:POKE 55296,14` writes the "A" glyph to screen memory and sets its color.

- Explanation of instructions used:
  - `LDA #$01` — Load immediate value $01 into the accumulator.
  - `STA $0400` — Store accumulator to absolute address $0400 (screen memory start; decimal 1024).

## Key Registers

- $0400 - Screen memory start (default top-left character cell; decimal 1024)
- $D800 - Color RAM start (color nibble for top-left cell; decimal 55296)
